# 运行与测试

## 本地运行

```bash
npm install
npm run dev
```

访问：
- 新版 Next UI：`http://127.0.0.1:8888`
- 老版 Express UI（功能兜底）：`http://127.0.0.1:8889`

## 生产构建（新版前端）

```bash
npm run build
```

说明：在当前环境中，`next build` 默认 Turbopack 可能受限，因此已固定为 `next build --webpack`。

## 语法检查（后端/旧前端）

```bash
for f in $(find src public/js -type f -name '*.js' | sort); do
  node --check "$f" || exit 1
done
```

## API 冒烟测试（含真实数据）

说明：以下命令直接测试老版 API 服务（`8889`）。

```bash
PORT=8889 HOST=127.0.0.1 node src/server.js >/tmp/peng_stock_server_8889.log 2>&1 &
SERVER_PID=$!
sleep 2

# 1) 健康检查
curl -s http://127.0.0.1:8889/api/health

# 2) 实时分析（真实行情源优先）
curl -s -X POST http://127.0.0.1:8889/api/v1/analysis \
  -H 'Content-Type: application/json' \
  -d '{"stockList":"AAPL,600519","async":false}'

# 3) 大盘复盘
curl -s 'http://127.0.0.1:8889/api/v1/market/review?region=both'

# 4) 智能导入（文本）
curl -s -X POST http://127.0.0.1:8889/api/v1/stocks/parse-import \
  -H 'Content-Type: application/json' \
  -d '{"text":"贵州茅台\nAAPL Apple"}'

# 5) 新增期货分类
curl -s -X POST http://127.0.0.1:8889/api/v1/futures/categories \
  -H 'Content-Type: application/json' \
  -d '{"name":"贵金属","description":"有色期货实时观察"}'

# 6) 分类下新增期货品种
curl -s -X POST http://127.0.0.1:8889/api/v1/futures/symbols \
  -H 'Content-Type: application/json' \
  -d '{"categoryId":1,"name":"白银主连","quoteCode":"101.SI00Y"}'

# 7) 期货监测（真实数据，含价格K线+成交量）
curl -s 'http://127.0.0.1:8889/api/v1/futures/monitor?timeframe=60m&limit=120'

# 8) 同步股票基础数据（A/HK/US，fail-open）
curl -s -X POST http://127.0.0.1:8889/api/v1/stock-basics/sync

# 可关注返回中的 aFundamentals.quality 覆盖率统计（A股字段补全质量）
# 首次全量补齐可能需要 20~60 秒（取决于外部接口响应）

# 9) 检索股票基础数据
curl -s 'http://127.0.0.1:8889/api/v1/stock-basics?market=US&q=AAPL&limit=10'

# 10) 查看个股基础信息详情（含总股本/流通股/主营业务/营业范围等）
curl -s 'http://127.0.0.1:8889/api/v1/stock-basics/000002?market=A'

# 10) 股票监测时间粒度
curl -s 'http://127.0.0.1:8889/api/v1/stock-monitor/timeframes'

# 11) 股票监测看板
curl -s 'http://127.0.0.1:8889/api/v1/stock-monitor/monitor?timeframe=1m&limit=120'

# 12) 回测
curl -s -X POST http://127.0.0.1:8889/api/v1/backtest/run \
  -H 'Content-Type: application/json' \
  -d '{"evaluationDays":5,"force":true}'

# 13) UI 资源
curl -s http://127.0.0.1:8889/ | head -n 20
curl -s http://127.0.0.1:8889/js/app.js | head -n 20

kill $SERVER_PID
```

## 测试关注点

- 后端服务：可启动、端口 `8889` 生效
- 分析 API：返回 `technical/news/fundamental_context/strategy/market_review`
- 期货监测 API：返回 `quote/candles/error`，且 `candles` 包含价格与成交量字段
  - 若 `push2his` 不可用，允许出现 `warning`，并从 `futsseapi.mx` 聚合K线（`candleDataSource=eastmoney.futsseapi.mx`）
- 股票基础数据 API：`sync` 返回 `markets/failedMarkets/failOpen`
  - 支持多源回退（A: 新浪财经，HK: HKEX 官方清单，US: Nasdaq Trader）
  - 服务启动后会自动做一次初始化同步（库为空时）
- 股票监测 API：返回 `timeframe/categories/items`，并支持 `1m/1d/1w/1M`
  - 若分钟线不可用，允许返回 `warning` 并降级到历史线（`candleDataSource=stockData.*`）
- 导入 API：返回 `confidence/confidenceLevel`
- 回测 API：返回 `directionHitRate/takeProfitHitRate/stopLossHitRate`
- UI：
  - 新版 Next 页面可访问（`/`、`/market`、`/import`、`/agent`、`/portfolio`、`/base-data`、`/stock-monitor`、`/stock/[code]`）
  - 新版通过 rewrite 访问 API（`http://127.0.0.1:8888/api/...` -> `8889`）
  - `stock/[code]` 页面支持服务端动态 metadata（SEO）
  - 股票监测表支持虚拟滚动，含 1000 行压力模式
  - 旧版 UI（`8889`）仍可访问作为功能兜底

## 常见问题

- 若启动时报错 `better-sqlite3 ... NODE_MODULE_VERSION`，请在当前 Node 版本下执行：

```bash
npm rebuild better-sqlite3
```

- 若需严格离线/内网部署：本项目前端已移除 CDN 外链（包括旧版页面中的 Google Fonts）。
