# 架构设计

## 1. 分层结构

- `routes`：定义 API 路由与 URL 规范
- `controllers`：处理 HTTP 输入输出
- `services`：承载核心业务逻辑
- `repositories`：数据访问与 SQL 操作
- `db`：数据库初始化、建表、迁移
- `middlewares`：认证、错误处理
- `events`：SSE 任务流广播

## 2. 模块说明

### 分析模块

- `analysisService`：统一调度同步/异步分析
- `taskService`：队列执行与任务状态管理
- `analysisEngineService`：
  - 技术指标计算
  - 新闻情绪融合
  - 筹码分布生成
  - `fundamental_context` 聚合
  - 市场策略输出
  - 决策报告生成
- `taskStream`：SSE 推送任务生命周期事件

### 市场模块

- `marketReviewService`：`cn/us/both` 复盘聚合
- `marketController`：复盘查询入口

### 数据模块

- `stockDataService`：行情获取
  - 主路径：Yahoo Finance
  - 次路径：Stooq 历史行情
  - 降级路径：Synthetic 合成行情（确定性）
- `newsIntelService`：新闻抓取与情绪评分
- `fundamentalService`：基本面结构化上下文生成（fail-open）

### 智能导入模块

- `importService`：统一处理文本/CSV/Excel/图片
- `nameResolverService`：名称转代码（本地字典 + 拼音 + 远程建议）
- `visionExtractService`：图片提取（可配置 Vision API，失败兜底）

### 回测模块

- `backtestService`：
  - 方向命中
  - 止盈命中
  - 止损命中
  - 分股票/总体统计
- `analysisRepository`：回测指标入库与统计查询

### 持仓模块

- `portfolioService`：账户、交易、现金流水、快照、风险
- `portfolioRepository`：持仓相关数据持久化

### 问股模块

- `agentService`：策略清单、问答生成、会话管理
- `chatRepository`：会话与消息存储

### 系统模块

- `systemConfigService`：配置读取、更新、校验
- `authService`：JWT 会话、认证开关、密码管理
- `notificationService`：SMTP 邮件通知（分析完成后发送）

## 3. 数据存储

SQLite 核心表：

- `users`
- `auth_settings`
- `system_configs`
- `analysis_tasks`
- `analysis_history`
- `backtest_results`（含 `tp_hit/sl_hit`）
- `portfolio_accounts`
- `portfolio_trades`
- `portfolio_cash_ledger`
- `portfolio_corporate_actions`
- `chat_sessions`
- `chat_messages`
- `usage_logs`

## 4. 关键设计决策

- 任务异步化：内存队列 + DB 状态持久化
- 前后端解耦：前端完全通过 REST API 调用
- fail-open：新闻/基本面/复盘异常不阻断主分析
- 数据可用性优先：行情失败时自动降级
- 低门槛部署：单进程 + SQLite
