# 需求文档

## 1. 项目目标

在 `Node.js + Express` 技术栈下，重构参考项目 `daily_stock_analysis` 的核心业务能力，保持原有功能并提供可维护的分层实现。

目标：

- 保留并补齐参考项目主功能链路（分析、复盘、导入、历史、回测、问股、持仓、系统配置）
- 默认端口使用 `8888`
- 支持邮件通知（SMTP），并预留其它通知渠道扩展位
- Web UI 采用现代化重构风格，按“模块 -> 子功能”组织
- 在外部数据源不稳定时保持 fail-open 可用

## 2. 功能需求（按模块）

### 2.1 AI

- 决策仪表盘：一句话核心结论
- 输出精确买入/止损/目标点位
- 输出可执行检查清单

### 2.2 分析

- 多维度分析：
  - 技术面（MA5/MA10/MA20、趋势判定）
  - 筹码分布（近120日成交量按价格区间聚合）
  - 舆情情报（新闻标题情绪分）
  - 实时行情（A/H/US）

### 2.3 市场

- 全球市场支持：A股、港股、美股及美股指数（`SPX`、`DJI`、`IXIC`）
- 大盘复盘接口：`cn` / `us` / `both`

### 2.4 基本面

- 输出 `fundamental_context` 结构化聚合：
  - `valuation`
  - `growth`
  - `earnings.data.financial_report`
  - `earnings.data.dividend`
  - `institution`
  - `capital_flow`
  - `dragon_tiger`
  - `boards`（含板块涨跌榜）
- 主链路 fail-open：子模块异常不阻断分析结果落库

### 2.5 策略

- A股：三段式复盘策略（进攻/均衡/防守）
- 美股：Regime Strategy（risk-on/neutral/risk-off）
- 必须附带“仅供参考，不构成投资建议”

### 2.6 智能导入

- 支持文本、CSV/Excel、图片导入
- Vision LLM 提取（可配置）+ 文件名兜底
- 名称 -> 代码解析（本地字典 + 拼音 + 远程建议）
- 置信度分层（high/medium/low）

### 2.7 历史与回测

- 历史记录：多选、全选、批量删除
- 回测：方向胜率、止盈命中率、止损命中率、平均收益

### 2.8 Agent 问股

- 多轮会话
- 11 种内置策略展示与问答调用
- Web/API 全链路可用

### 2.9 有色期货实时监测

- 新增独立监测页面（菜单中文化）
- 支持新增分类（如贵金属、工业金属等）
- 支持在分类下新增期货品种（代码支持 `101.SI00Y` / `SI00Y`）
- 每个品种展示：
  - 价格K线图（OHLC）
  - 成交量图
  - 最新价、涨跌幅、持仓量、成交量
- 支持多时间粒度切换：`1m / 5m / 15m / 30m / 60m / 1d / 1w / 1M`
- 数据源来自东方财富全球期货链路（`futsseapi` + `push2his`），单品种失败不阻断整体看板（fail-open）

### 2.10 基础数据（股票数据）

- 菜单位置：`系统配置 -> 基础数据 -> 股票数据`
- 支持手动同步三大市场股票基础数据（A/HK/US）
- 支持按代码、名称、市场检索基础数据
- 支持“本地基础数据 + 远程实时行情”的组合详情视图
- 同步链路 fail-open：单市场拉取失败时保留其它市场结果并返回失败明细
- 多源同步机制（不依赖单一来源）：
  - A股：东方财富失败时自动回退新浪财经节点源
  - 港股：东方财富失败时自动回退 HKEX 官方证券清单
  - 美股：东方财富失败时自动回退 Nasdaq Trader 官方名单
- 服务启动时自动执行一次初始化同步；“手动同步”用于后续查漏补缺

### 2.11 股票实时监测

- 菜单位置：`行情中心 -> 股票监测`
- 支持分类管理：新增、编辑、删除
- 支持股票池管理：新增、删除，支持从基础数据下拉快速选择或手动输入
- 每个股票卡片展示：
  - 最新价、涨跌、涨跌幅、成交量、最后刷新时间
  - 价格图 + 成交量图
- 支持时间粒度切换：`1m / 1d / 1w / 1M`
- 默认长周期图查询最近 `100` 个时间点
- 自动刷新默认 `30` 秒，可切换其它间隔或关闭
- 分钟数据不可用时，自动降级到历史线并给出提示（fail-open）

## 3. 非功能需求

- 可维护性：严格分层（routes/controllers/services/repositories）
- 可扩展性：通知、数据源、策略引擎可增量扩展
- 可用性：Yahoo -> Stooq -> Synthetic 降级链路
- 数据持久化：SQLite 单机部署
