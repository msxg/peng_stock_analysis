# 16 A股平均股价指标模块设计

## 1. 文档定位

本文档定义“A股平均股价指标中心”模块。

本模块是独立系统指标模块，负责每日产出并提供可追溯的市场统计指标。

V1 目标指标：

- A股平均股价
- A股中位数股价

## 2. 范围与边界

### 2.1 V1 范围

- 仅 A股范围
- 支持规则定义（口径定义）
- 支持按交易日计算指标
- 支持指标查询与审计追溯

### 2.2 明确边界

- 不作为“股票代码”参与行情查询
- 不依赖第三方软件口径，口径由系统显式配置
- 不负责上层业务策略解释，仅负责指标计算与提供

## 3. 核心概念

### 3.1 规则（Rule）

规则用于定义“如何计算平均/中位数”。

V1 规则能力：

- 样本范围（`scope_key`，V1 固定 `ALL_A`）
- 价格字段（默认“不复权收盘价”，支持选择）
- 是否剔除当日停牌股票（支持开关，默认剔除）
- 新股过滤规则（支持“上市未满 N 个交易日剔除”）
- 是否包含 ST 股票（支持开关，默认包含）
- 最低有效样本数
- 启停状态
- 默认规则标记（同一 `scope_key` 下有且仅有一条默认）

### 3.2 指标快照（Daily Metric）

按“交易日 + 规则”生成的结果快照。

每条快照可追溯：

- 使用了哪条规则
- 规则版本/规则 key
- 输入样本数
- 平均值、中位数
- 计算时间

## 4. 规则字段设计

### 4.1 关键字段

建议字段：

- `scope_key`：样本范围，V1 默认 `ALL_A`
- `price_mode`：价格口径，默认 `close_raw`
- `exclude_suspended`：是否剔除当日停牌，默认 `1`
- `min_listing_trading_days`：上市最少交易日门槛，默认 `0`
- `include_st`：是否包含 ST，默认 `1`
- `min_sample_size`：最低有效样本数
- `is_enabled`：是否启用
- `is_default`：是否默认规则（同一 `scope_key` 下唯一）

### 4.2 价格字段口径

V1 默认：

- `close_raw`：不复权收盘价

V1 预留可选：

- `close_qfq`：前复权收盘价
- `close_hfq`：后复权收盘价

说明：当本地数据未提供复权口径时，可暂不开放对应选项，仅保留规则字段与扩展位。

### 4.3 新股过滤规则

规则使用 `min_listing_trading_days` 控制。

- `0`：不剔除新股
- `60`：剔除上市未满 60 个交易日的新股

上市交易日数以本地日线数据中“该标的在目标交易日前的有效交易日计数”为准。

## 5. 数据模型设计

### 5.1 `a_share_market_metric_rules`

建议字段：

- `id`
- `rule_key`（唯一）
- `name`
- `scope_key`
- `price_mode`
- `exclude_suspended`
- `min_listing_trading_days`
- `include_st`
- `min_sample_size`
- `is_enabled`
- `created_at`
- `updated_at`

### 5.2 `a_share_market_metrics_daily`

建议字段：

- `id`
- `trade_day`
- `rule_id`
- `rule_key_snapshot`
- `scope_key`
- `price_mode`
- `avg_price`
- `median_price`
- `sample_size`
- `source_dataset`（默认 `stock_eod_bars`）
- `computed_at`
- `created_at`

唯一约束建议：

- `(trade_day, rule_id, scope_key)`

## 6. 分层与模块拆分

### 6.1 Service

建议新增：

- `aShareMarketMetricService`
- `aShareMarketMetricQueryService`

职责：

- 规则读取
- 计算编排
- 指标快照落库
- 指标查询 DTO 组装

### 6.2 Repository

建议新增：

- `aShareMarketMetricRuleRepository`
- `aShareMarketMetricDailyRepository`

职责：

- 规则 CRUD
- 每日指标 upsert
- 按日期/规则查询

### 6.3 Job

建议新增每日任务入口：

- 支持手动触发
- 支持定时触发
- 支持补算历史交易日

## 7. 计算流程

1. 读取启用规则
2. 确定目标交易日
3. 从 `stock_eod_bars` 抽取 A股样本
4. 按规则执行过滤：
   - 停牌过滤（开关）
   - 新股交易日门槛过滤
   - ST 包含开关过滤
5. 计算平均值与中位数
6. 写入 `a_share_market_metrics_daily`
7. 产出任务回执

## 8. 口径与一致性

- V1 `scope_key` 固定 `ALL_A`
- 默认使用“不复权收盘价”计算，可按规则切换价格口径
- 所有指标结果必须带 `trade_day + rule_id`
- 任意时点可重算并覆盖同主键快照（保持幂等）
- 默认规则必须存在且保持唯一（V1 全局即全A仅一条）

## 9. 对外使用方式（其它应用模块）

本模块对上层模块提供“只读指标消费”，不要求上层感知计算细节。

推荐两种使用方式：

- API 查询：通过 `market-metrics` 接口按交易日读取指标
- 内部服务调用：通过 `aShareMarketMetricQueryService` 读取标准 DTO

上层模块使用建议：

- 行情看板：展示每日平均/中位数并标注口径与样本数
- 策略模块：读取指定规则的当日指标作为市场宽度参考
- 报表模块：按日期范围拉取指标序列进行对比分析

应用调用约定：

- 查询时建议允许不传 `ruleKey`
- 不传 `ruleKey` 时，系统自动按默认规则返回数据
- 传入 `ruleKey` 时，按指定规则返回
- 默认规则变更由规则中心统一维护，应用侧不应硬编码规则标识

返回 DTO 建议包含：

- `tradeDay`
- `ruleKey`
- `scopeKey`
- `priceMode`
- `avgPrice`
- `medianPrice`
- `sampleSize`
- `computedAt`

## 10. 接口草案

### 10.1 规则管理

- `GET /api/v1/market-metrics/rules`
- `POST /api/v1/market-metrics/rules`
- `PUT /api/v1/market-metrics/rules/:ruleId`

### 10.2 指标查询

- `GET /api/v1/market-metrics/daily?tradeDay=2026-05-12&scopeKey=ALL_A`
- `GET /api/v1/market-metrics/daily/range?startDay=2026-01-01&endDay=2026-05-12`

说明：

- `ruleKey` 可选；未传时自动使用默认规则

### 10.3 任务触发

- `POST /api/v1/market-metrics/compute`

请求示例：

```json
{
  "tradeDay": "2026-05-12",
  "ruleKey": "ALL_A_CLOSE_RAW_V1",
  "force": false
}
```

## 11. 错误处理与 fail-open

- 数据缺失或样本不足：
  - 不写入错误值
  - 返回明确错误/警告
- 多规则并行时：
  - 单规则失败不阻断其他规则
  - 回执按规则给出成功/失败明细

### 12.1 验收点

- 能按规则对指定交易日产出平均/中位数
- 停牌过滤开关可用
- 新股交易日门槛（如 60）可用
- ST 包含开关可用
- 能按日期范围查询历史指标
- 能保证幂等写入
- 能追溯到规则版本

### 12.2 测试建议

- Repository 单测：规则与快照 CRUD
- Service 单测：样本过滤与统计计算
- API 集成测试：规则管理、计算触发、结果查询
