# 15 条件选股模块设计

## 1. 文档定位

本文档定义“条件选股”模块（V1）的功能边界、分层方案、接口结构与数据流。

本模块定位为：

- A股股票池筛选工具
- 与现有“行情监测 / 蓝筹模式标的池”联动的结果生产入口
- 支持结构化条件表达与批量动作

## 2. 范围与边界

### 2.1 V1 范围

- 仅支持 A股
- 支持市场细分筛选：`SH/SZ/BJ`（默认全选）
- 支持板块细分筛选：`MAIN/GEM/STAR`（默认全选）
- 支持日期范围的行情条件
- 支持比较型条件表达式
- 支持批量结果动作：
  - 一键加入监控分类（`monitor_symbols`）
  - 一键加入蓝筹标的池（`bluechip_pool_symbols`）

### 2.2 明确不在 V1 范围

- 不支持港股/美股
- 不支持 OR 组合条件组（V1 仅 AND）
- 不支持把“平均股价/中位数股价”作为可选条件项（仅预留扩展位）

### 2.3 与“平均股价指标中心”的关系

- 条件选股与平均股价指标中心完全解耦
- V1 条件选股不依赖平均股价指标中心
- 后续仅通过“系统指标读取”方式做弱依赖扩展

## 3. 业务目标

- 统一筛选入口：把基础面与行情条件在本地数据内完成筛选
- 提高运营效率：筛选结果可直接沉淀到监控分类和蓝筹池
- 提供可解释性：结果返回命中条件摘要、数据日期与 warning

## 4. 条件模型设计

### 4.1 顶层筛选请求

```json
{
  "market": "A",
  "subMarkets": ["SH", "SZ", "BJ"],
  "boardSegments": ["MAIN", "GEM", "STAR"],
  "dateRange": {
    "startDate": "2026-01-01",
    "endDate": "2026-05-12"
  },
  "fundamentals": {
    "totalMarketCapMin": 10000000000,
    "totalMarketCapMax": 500000000000,
    "listingDaysMin": 120
  },
  "technicalRules": [
    {
      "left": { "type": "field", "name": "close" },
      "operator": ">",
      "right": { "type": "indicator", "name": "ma20" }
    },
    {
      "left": { "type": "metric", "name": "pctChgN", "args": { "days": 20 } },
      "operator": ">=",
      "right": { "type": "const", "value": 5 }
    }
  ],
  "sort": {
    "field": "pctChgN",
    "order": "desc"
  },
  "page": 1,
  "limit": 50
}
```

### 4.2 比较表达式

统一为：`left operand operator right operand`

- `operator`：`> >= < <= == !=`
- `left/right` 值类型：
  - `field`：`close/high/low/amount/vol`
  - `indicator`：`ma5/ma10/ma20/ma60`
  - `metric`：`pctChgN/rangeAvgClose/rangeMaxClose/rangeMinClose`
  - `const`：常数

### 4.3 代码板块识别规则（V1）

- `GEM`：`SZ300xxx`、`SZ301xxx`
- `STAR`：`SH688xxx`
- `MAIN`：其余 A股代码（含 BJ）

优先按代码规则识别，避免依赖文本行业字段。

## 5. 结果能力设计

### 5.1 查询结果返回

- 分页结果列表
- 结果总数
- 数据基准日（`dataAsOf`）
- 条件命中摘要（可用于前端展示）
- fail-open warning

### 5.2 批量动作一：加入监控分类

目标表：`monitor_symbols`

设计原则：

- 支持批量代码入参
- 自动去重（同分类、同类型、同市场、同代码）
- 返回回执：`success/skipped/failed`

### 5.3 批量动作二：加入蓝筹标的池

目标表：`bluechip_pool_symbols`

设计原则：

- 支持批量代码入参
- 自动去重（同 pool 下相同 stock_code）
- 返回回执：`success/skipped/failed`

## 6. 分层与模块拆分

### 6.1 Route / Controller

建议新增：

- `src/controllers/stockScreeningController.js`
- 路由前缀：`/api/v1/stock-screening`

接口：

- `POST /query`
- `POST /actions/add-to-monitor`
- `POST /actions/add-to-bluechip-pool`

### 6.2 Service

建议新增：

- `src/services/stockScreeningService.js`

职责：

- 参数标准化与校验
- 子市场/板块规则解析
- 调用 repository 执行筛选
- 组装结果 DTO
- 批量动作编排

### 6.3 Repository

建议新增：

- `src/repositories/stockScreeningRepository.js`

职责：

- 基础条件 SQL 过滤
- 日期范围内行情数据提取
- 返回原始记录与计算所需字段

约束：

- 不做页面文案拼装
- 不做业务策略解释
- 不在 SQL 中混入页面分支逻辑

## 7. 数据流与执行策略

### 7.1 查询流程

1. 校验并标准化请求参数
2. 执行“基础条件 + 市场/板块预过滤”
3. 候选集读取日期范围内 EOD 数据
4. 计算技术指标与比较条件
5. 过滤命中集合并排序分页
6. 返回结果与 warning

### 7.2 性能策略

- 先过滤再计算，避免全市场逐只全量计算
- 限制单次最大候选规模（例如 4000）
- 限制最大日期跨度（例如 2 年）
- 指标计算窗口不足时，该标的记 warning 并剔除或按策略跳过

## 8. 接口草案

### 8.1 查询

- `POST /api/v1/stock-screening/query`

响应示例：

```json
{
  "page": 1,
  "limit": 50,
  "total": 382,
  "dataAsOf": "2026-05-12",
  "items": [
    {
      "code": "SZ300750",
      "name": "宁德时代",
      "subMarket": "SZ",
      "boardSegment": "GEM",
      "close": 188.25,
      "ma20": 180.14,
      "pctChgN": 8.32,
      "hitRules": [
        "close > ma20",
        "pctChgN(20) >= 5"
      ]
    }
  ],
  "warning": null,
  "failOpen": true
}
```

### 8.2 批量加入监控分类

- `POST /api/v1/stock-screening/actions/add-to-monitor`

### 8.3 批量加入蓝筹池

- `POST /api/v1/stock-screening/actions/add-to-bluechip-pool`

两类动作通用回执建议：

```json
{
  "success": 12,
  "skipped": 3,
  "failed": 1,
  "details": [
    { "code": "600519", "status": "success" },
    { "code": "000001", "status": "skipped", "reason": "duplicate" },
    { "code": "300999", "status": "failed", "reason": "not_found" }
  ]
}
```

## 9. 错误与 fail-open 约定

- 条件非法：返回 `400`
- 日期范围非法：返回 `400`
- 部分标的指标不足：不阻断全局，写入 `warning`
- 批量动作单条失败：不中断整批，按明细回执

## 10. 前端交互建议

页面建议：`行情中心 -> 条件选股`

- 左侧：筛选条件区
- 右侧：结果表
- 顶部：执行筛选、重置条件
- 表格：勾选、批量加入监控、批量加入蓝筹池
- 交互提示：互斥条件直接提示“结果为空”而非报错

## 11. 测试与验收

### 11.1 核心验收点

- `subMarkets` 默认全选生效
- `boardSegments` 默认全选生效
- 日期范围条件可用
- 比较表达式可用
- 批量加入监控分类可用
- 批量加入蓝筹池可用
- 重复写入被正确跳过

### 11.2 建议测试层次

- Service 单测：条件解析与规则求值
- Repository 单测：关键过滤 SQL
- API 集成测试：查询 + 两个批量动作
