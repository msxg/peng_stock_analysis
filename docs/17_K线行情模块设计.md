# 17 K线行情模块设计

## 1. 模块定位

`/kline` 是一个独立的行情中心入口，用于查询单个品种的K线行情。

## 2. 页面形式

- `/kline`：手动输入查询入口
- `/kline/[code]`：直达详情页，适合从其它页面跳转

两者共用同一套核心组件与数据逻辑。

## 3. 页面结构

### 3.1 输入区

仅保留：

- 分类
- 代码输入
- 查询按钮

### 3.2 概览区

按以下层级展示：

- 品种名称、代码
- 最新价、涨跌额、涨跌幅
- 其它价格与交易数据

### 3.3 图表区

- K线图
- 图表上方时间粒度切换
- warning / error 提示

## 4. 时间粒度规则

- 股票：支持 `分时(1m) / 1d / 1w / 1M`，默认 `分时`
- 期货：使用标准粒度切换
- 时间粒度按钮放在图表上方，点击后立即重新查询并刷新图表

补充：

- 股票 `分时(1m)` 走独立分钟K线链路（非日线聚合），用于展示实时分钟行情
- 股票 `1d/1w/1M` 继续走日线历史链路，`1w/1M` 由日线聚合生成

## 5. 直达规则

- `/kline/600519` 可直接打开股票详情
- `/kline/101.GC00Y` 可直接打开期货详情
- 页面会根据代码自动判断股票或期货，并执行相应查询

## 7. 数据链路说明

股票：

- 分时：`GET /stock-monitor/kline?stockCode=600519&timeframe=1m&limit=1800`
- 日/周/月：`GET /stocks/:stockCode/history?days=...` + 前端聚合
- 概览实时价优先使用分钟链路返回的 `quote`，报价失败时用分钟K线末值兜底

期货：

- `GET /futures/monitor?quoteCode=...&timeframe=...&limit=...`
## 8. 模块结论

这个页面是单品种行情入口，不承接批量监测职责，只负责查询、展示和直达。

## 9. 概览字段展示规则补充（2026-05-15）

针对股票分时场景，概览面板补充如下规则：

- 右上角显示 `涨停 / 跌停 / 振幅`，左侧指标区不重复展示这三项
- K线区不再展示 `MAX / MIN` 统计行，仅保留时间粒度按钮与图形本体

`涨停/跌停` 取值策略：

1. 优先使用实时接口直接返回的 `quote.limitUp / quote.limitDown`
2. 若接口未返回，则按 `昨收` 与板块涨跌幅限制前端兜底计算

前端兜底计算规则（A股）：

- `ST` 股票：`5%`
- 创业板/科创板（代码前缀 `300/301/688/689`）：`20%`
- 北交所（`BJ` 或代码前缀 `4/8`）：`30%`
- 其它A股：`10%`

计算公式：

- `涨停 = round(prevClose * (1 + limitPct), 2)`
- `跌停 = round(prevClose * (1 - limitPct), 2)`

`振幅` 取值策略：

1. 优先使用实时接口返回的 `quote.amplitude`
2. 若接口未返回，则按 `(最高 - 最低) / 昨收 * 100` 计算
